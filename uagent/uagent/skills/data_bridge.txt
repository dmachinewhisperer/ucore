# Bridging Data Between Device and Python

## Pattern: Device prints, Python parses

The device (%%ucore) and Python run in separate processes. Data passes through cell output (stdout).

### Step 1: Device collects and prints

```python
%%ucore
import json
readings = [23.1, 23.4, 23.2]
print(json.dumps(readings))
```

### Step 2: Python reads the output

After running the device cell and reading its output via read_notebook_cells,
the agent extracts the printed data and injects it into a Python cell:

```python
import json
raw = '[23.1, 23.4, 23.2]'  # copied from device cell output
data = json.loads(raw)
```

## Rules

- Always use `json.dumps()` on the device for structured data.
- For simple values, `print()` is fine. For lists/dicts, use JSON.
- The agent reads cell output via `read_notebook_cells` and extracts the printed value.
- Large datasets: collect on device, print as JSON array. Parse in Python.
- Binary data: not supported over stdout. Convert to numeric lists on device.

## Common gotcha

MicroPython's `json` module exists but has limits. For very large data,
print comma-separated values and parse with `str.split()` in Python:

```python
%%ucore
readings = [adc.read() for _ in range(1000)]
print(",".join(str(r) for r in readings))
```

```python
raw = "2048,2051,2049,..."  # from cell output
data = [int(x) for x in raw.split(",")]
```
